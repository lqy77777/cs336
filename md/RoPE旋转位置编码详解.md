# RoPE(Rotary Positional Embedding)详解

> 依据 `cs336_assignment1_basics.pdf` 3.4.3 节。本文件只讲**原理和推理过程**,不含任何实现代码——具体怎么写,是 `rope` 这道题(2 分)要你自己完成的部分。

## 1. 为什么需要位置编码

Attention 这个操作本身,对输入序列的**顺序不敏感**——把 $Q$、$K$、$V$ 里的 token 打乱重排,attention 算出来的加权求和结果只是跟着重排了一下,数值本身没有变化(这是 attention 公式 $\text{softmax}(QK^\top/\sqrt{d_k})V$ 的一个数学性质)。也就是说,如果不额外注入"每个 token 在序列里第几个位置"这个信息,模型**分不清"猫追狗"和"狗追猫"**——两句话的 token 集合一样,只是顺序不同,而 attention 本身"看不见"顺序。位置编码就是用来把这个缺失的信息补回去的。

## 2. RoPE 的核心思想:用"旋转"编码位置,而不是"相加"

原始 Transformer 论文的做法是把一个和位置相关的向量**加**到 embedding 上。RoPE 换了一种方式:**对 query/key 向量做一次和位置相关的旋转**,而不是加法。

具体来说,给定位置 $i$ 处的 query 向量 $q^{(i)} = W_q x^{(i)} \in \mathbb{R}^d$,RoPE 应用一个**依赖位置 $i$ 的旋转矩阵 $R^i$**:

$$
q'^{(i)} = R^i q^{(i)}
$$

$R^i$ 是一个**正交矩阵**(旋转变换),它不改变向量的长度,只改变向量的"方向"——方向改变的角度,取决于这个 token 在序列里的位置 $i$。

## 3. 具体怎么"旋转":把向量拆成一对一对的 2D 向量

$R^i$ 不是对整个 $d$ 维向量做一次笼统的旋转,而是把这 $d$ 个分量**两两分组**,每一组当作一个独立的 2D 平面向量单独旋转:

$$
(q_1, q_2),\ (q_3, q_4),\ \ldots,\ (q_{d-1}, q_d)
$$

一共 $d/2$ 组。第 $k$ 组($k \in \{1, \ldots, d/2\}$)对应的分量 $q^{(i)}_{2k-1:2k}$,被当成一个 2D 向量,按下面这个角度旋转:

$$
\theta_{i,k} = \frac{i}{\Theta^{(2k-2)/d}}
$$

$\Theta$ 是一个常数超参数(一般取一个较大的数,比如 10000)。**每一对分量的旋转角度,是"位置 $i$"和"这一对属于第几组($k$)"共同决定的**——同一个位置 $i$,不同的组 $k$ 会有不同的旋转角度。

对第 $k$ 组,2D 旋转用的是标准的二维旋转矩阵:

$$
R_k^i =
\begin{pmatrix}
\cos(\theta_{i,k}) & -\sin(\theta_{i,k}) \\
\sin(\theta_{i,k}) & \cos(\theta_{i,k})
\end{pmatrix}
$$

（这就是平面几何里"把一个二维向量绕原点旋转 $\theta$ 角度"的标准公式。）

**注意:这要求 $d$(也就是 `d_k`,query/key 的维度)必须是偶数**——否则没法两两配对。

## 4. 完整的旋转矩阵:分块对角矩阵

把 $d/2$ 个 $R_k^i$ 按顺序排在对角线上、其余位置全部填 0,就得到完整的 $d \times d$ 旋转矩阵:

$$
R^i =
\begin{pmatrix}
R_1^i & 0 & 0 & \cdots & 0 \\
0 & R_2^i & 0 & \cdots & 0 \\
0 & 0 & R_3^i & \cdots & 0 \\
\vdots & & & \ddots & \vdots \\
0 & 0 & 0 & \cdots & R_{d/2}^i
\end{pmatrix}
$$

每个 $0$ 代表一个 $2\times 2$ 的零矩阵。这个矩阵结构上是"分块对角"——除了对角线上那些 $2\times 2$ 小旋转块,其余全是 0。

## 5. 为什么每一对(每个 $k$)的旋转角速度不一样

看一下角度公式 $\theta_{i,k} = i / \Theta^{(2k-2)/d}$:分母 $\Theta^{(2k-2)/d}$ 随着 $k$ 增大而**增大**(因为 $\Theta > 1$,指数 $(2k-2)/d$ 随 $k$ 增大)。分母越大,同样的位置 $i$ 算出来的角度 $\theta_{i,k}$ 就越小。

也就是说:

- **$k$ 小的那几对**(高频):位置每往后挪一步,角度变化**大**——旋转得快
- **$k$ 大的那几对**(低频):位置每往后挪一步,角度变化**小**——旋转得慢

这个设计让模型同时拥有"变化剧烈、能分辨相邻位置差异"和"变化缓慢、能覆盖很远距离"这两种不同粒度的位置信息——和原始 Transformer 论文用不同频率的正弦/余弦函数编码位置,是同一个思路的不同实现方式。

## 6. RoPE 最关键的性质:天然编码"相对位置"

这是 RoPE 相比"直接把位置向量加到 embedding 上"这种做法,最值得理解的优势:

因为旋转矩阵满足 $R^i$ 和 $R^j$ 的组合关系(旋转是可以叠加的),query 在位置 $i$ 旋转后、和 key 在位置 $j$ 旋转后,两者的点积(attention 算分数用的正是点积)只依赖于 **$i - j$(相对距离)**,不依赖 $i$ 和 $j$ 各自的绝对值。也就是说:两个 token 之间"隔了几步",比它们"具体在序列的第几个位置",对 attention 分数的影响更直接、更自然——这正是 RoPE 被称为"相对位置编码"的原因,即使它的构造方式(对每个位置单独应用一个旋转)看起来像是在编码"绝对位置"。

## 7. 应用范围

**只作用在 $Q$ 和 $K$ 上,不作用在 $V$ 上**——讲义原话:"RoPE should be applied to the query and key vectors, but not the value vectors"。这是因为 attention 的分数($QK^\top$)才是"哪个 token 应该关注哪个 token"这件事发生的地方,位置信息只需要影响"谁该看谁",不需要污染最终被加权求和的内容本身($V$)。

**多头场景下,"head"这个维度要当作一个额外的批量维**——讲义原话:"the head dimension should be handled as a batch dimension"。也就是说,每个 head 各自独立地对自己那一份 query/key 做**完全相同的旋转过程**(用同样的位置信息),head 之间互不影响,只是"批量地"重复同一套旋转逻辑。

**Key 向量的处理方式和 Query 完全一致**,只是用 key 自己的位置 $j$(而不是 query 的位置 $i$)去算对应的 $R^j$。

## 8. 实现效率提示(讲义原话)

- **不需要真的构造出完整的 $d\times d$ 矩阵**去做矩阵乘法——那样会浪费大量计算在乘 0 上面。一个好的实现应该利用"分块对角"这个结构,直接对一对一对的分量做旋转,而不是走完整矩阵乘法
- **$\cos(\theta_{i,k})$、$\sin(\theta_{i,k})$ 这些值,在同一序列内部、跨层、跨 batch 都是可以复用的**——它们只取决于位置 $i$ 和组号 $k$,和 query/key 的具体数值内容毫无关系,所以可以**只算一次、缓存起来**反复用
- 缓存这些 cos/sin 值,推荐用 `self.register_buffer(persistent=False)`,而**不是** `nn.Parameter`——因为这些值是固定算出来的,不需要被训练,也不需要存进模型的 checkpoint(可以随时用 `theta` 重新算出来)
- **这一层没有任何可学习参数**——`RotaryPositionalEmbedding` 是一个纯粹的、确定性的几何变换,不参与梯度更新

## 9. 推荐接口(讲义给出,仅接口本身)

```python
def __init__(self, theta: float, d_k: int, max_seq_len: int, device=None)
def forward(self, x: torch.Tensor, token_positions: torch.Tensor) -> torch.Tensor
```

- `theta`:公式里的常数 $\Theta$
- `d_k`:query/key 向量的维度
- `max_seq_len`:预计会输入的最大序列长度(决定了 cos/sin 缓存要预先算多大)
- `forward` 的输入 `x` 形状是 `(..., seq_len, d_k)`,要能容忍**任意数量的批量维**;`token_positions` 形状是 `(..., seq_len)`,标注 `x` 里每个 token 沿序列方向的具体位置
- 讲义特别提醒:要用 `token_positions` 去**索引/切片**预先算好的 cos/sin 张量(沿序列这一维)——不能想当然地假设位置就是 $0, 1, 2, \ldots, \text{seq\_len}-1$ 这种连续序列,而应该按 `token_positions` 里给定的实际值去取对应的旋转角度

## 10. 对比:和最开始(原始 Transformer)的位置编码有什么不同

### 10.1 原始位置编码是什么

原始 Transformer 论文(Vaswani et al., 2017)用的是**正弦/余弦位置编码(Sinusoidal Positional Encoding)**。对序列里第 $\text{pos}$ 个位置,构造一个 $d_{\text{model}}$ 维的向量 $PE(\text{pos})$,第 $2j$、$2j+1$ 两个分量分别是:

$$
PE(\text{pos}, 2j) = \sin\!\left(\frac{\text{pos}}{10000^{2j/d_{\text{model}}}}\right), \qquad
PE(\text{pos}, 2j+1) = \cos\!\left(\frac{\text{pos}}{10000^{2j/d_{\text{model}}}}\right)
$$

其中 $j = 0, 1, \ldots, d_{\text{model}}/2 - 1$。算出来的这个向量,直接**加**到对应位置的 token embedding 上:

$$
x'_{\text{pos}} = x_{\text{pos}} + PE(\text{pos})
$$

这一步只在**模型最底部、进入第一个 Transformer block 之前做一次**——位置信息被"注入"进去之后,就和内容混在同一个向量里,随着残差流逐层往上传。整个 $PE$ 是**固定算出来的**(不是训练学出来的),也没有任何可学习参数,这一点和 RoPE 是一致的。

### 10.2 逐项对比

| | 原始正弦/余弦编码 | RoPE |
| --- | --- | --- |
| **作用在哪一步** | 只在输入层加一次(最底部) | 每一个 attention 层都重新作用一次,直接作用在当层的 Q、K 上 |
| **和内容怎么结合** | **相加**(位置向量 + 内容向量,结果混在一起) | **旋转**(对内容向量做几何变换,不是叠加一个独立信号) |
| **可学习参数** | 无(固定公式算出) | 无(固定公式算出) |
| **编码的是绝对位置还是相对位置** | 直接编码**绝对位置**(每个位置对应唯一固定的向量) | 表面上按绝对位置 $i$ 旋转,但旋转的组合性质让 attention 的点积结果只依赖**相对距离** $i-j$ |
| **频率结构** | 不同维度对应不同频率的正弦/余弦波,频率按 $10000^{2j/d_{\text{model}}}$ 成几何级数分布 | 不同分组 $k$ 对应不同旋转速度,频率按 $\Theta^{(2k-2)/d}$ 成几何级数分布——和左边是**同一套"用 10000 这类大常数构造几何级数频率"的思路**,只是应用方式从"加法"换成了"旋转" |

### 10.3 两者最本质的差异

两种方法都用了"不同维度对应不同频率"这个共同的设计思路(这也是为什么 RoPE 的公式形式看起来和原始正弦编码很像,两者都在指数上用了同一个大常数 10000 类似的角色),但**注入位置信息的方式完全不同**:

- 原始编码是**加法式的、一次性的**——位置信息在最底层就和内容"焊"在一起,后面每一层都要靠模型自己去"记住"和"传递"这份位置信息,而且理论上"相对位置"信息虽然存在(三角函数的和角公式让 $PE(\text{pos}+k)$ 能表示成 $PE(\text{pos})$ 的线性组合),但这个性质没有被直接用在 attention 的计算过程里
- RoPE 是**乘法式(旋转)、逐层重新应用的**——每次算 attention 分数时,位置信息都通过旋转直接参与到 $QK^\top$ 这个点积计算本身,而且我们在第 6 节讲过,这个点积的结果**直接**只依赖相对距离,不需要模型"额外学会"这件事

这也是为什么现代大模型(LLaMA 等)普遍弃用了原始的加法式位置编码、转而采用 RoPE 这类方案——相对位置信息被更直接地建模进了 attention 机制本身,而不是依赖模型自己在训练中间接学出来。

## 11. 小结

RoPE 做的事情,一句话概括:**把 query/key 向量拆成若干对二维分量,每一对按位置相关的角度做旋转,不同分组的旋转速度不同(覆盖不同粒度的位置信息),而且旋转的组合性质让它天然编码了 token 之间的相对距离,而不只是绝对位置**——整个过程不含可学习参数,是一个纯几何变换。
