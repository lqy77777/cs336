# 1 作业概述（Assignment Overview）

> 译自 CS336 Assignment 1 (basics): Building a Transformer LM，Version 26.0.3，第 1 节。仅供个人学习使用。

在本次作业中，你将从零开始构建训练一个标准 Transformer 语言模型（LM）所需的全部组件，并实际训练一些模型。

## 你将实现的内容

1. 字节对编码（BPE）分词器（第 2 节）
2. Transformer 语言模型（LM）（第 3 节）
3. 交叉熵损失函数和 AdamW 优化器（第 4 节）
4. 训练循环，支持模型和优化器状态的序列化与加载（第 5 节）

## 你将运行的内容

1. 在 TinyStories 数据集上训练 BPE 分词器。
2. 用训练好的分词器对数据集进行编码，把它转换成整数 ID 序列。
3. 在 TinyStories 数据集上训练 Transformer 语言模型。
4. 用训练好的 Transformer 语言模型生成样本并评估困惑度（perplexity）。
5. 在 OpenWebText 上训练模型，并把你得到的困惑度提交到排行榜（leaderboard）。

## 你可以使用的工具

我们希望你从零开始构建每一个组件。特别地，你**不可以**使用 `torch.nn`、`torch.nn.functional` 或 `torch.optim` 中的任何定义，但以下内容除外：

- `torch.nn.Parameter`
- `torch.nn` 中的容器类（例如 `Module`、`ModuleList`、`Sequential` 等）[^1]
- `torch.optim.Optimizer` 基类

你可以使用 PyTorch 的任何其他定义。如果你想用某个函数或类，但不确定是否允许，欢迎在 Slack 上提问。拿不准的时候，想一想使用它是否违背了本作业「从零开始」的精神。

[^1]: 完整列表见 [pytorch.org/docs/stable/nn.html#containers](https://pytorch.org/docs/stable/nn.html#containers)。

## 代码结构

作业代码以及这份讲义都可以在 GitHub 上获取：

> [github.com/stanford-cs336/assignment1-basics](https://github.com/stanford-cs336/assignment1-basics)

请 `git clone` 这个仓库。如果有任何更新，我们会通知你，你可以 `git pull` 获取最新版本。

1. `cs336_basics/*`：这是你写代码的地方。注意这里面没有任何代码——你可以从零开始，想怎么写就怎么写！
2. `adapters.py`：这里规定了你的代码必须具备的一组功能。对于每项功能（例如缩放点积注意力 scaled dot product attention），只需通过调用你自己的代码来填写它的实现（例如 `run_scaled_dot_product_attention`）。注意：你对 `adapters.py` 的修改不应包含任何实质性的逻辑，这只是胶水代码（glue code）。
3. `test_*.py`：这里包含你必须通过的所有测试（例如 `test_scaled_dot_product_attention`），这些测试会调用 `adapters.py` 中定义的钩子。请不要修改测试文件。

### 低资源提示（Low-Resource Tip）：在 Apple Silicon 或 CPU 上完成作业 1

使用课程官方参考代码，我们可以在一台 36 GB 内存的 Apple M4 Max 芯片上训练出能生成相当流畅文本的语言模型：用 Metal GPU（MPS）不到 5 分钟，用 CPU 大约 30 分钟。如果这些词对你来说没什么概念，别担心！你只需要知道：只要你有一台还算新的笔记本电脑，并且你的实现正确且高效，你就能训练出一个能生成简单儿童故事、流畅度不错的小型语言模型。

在作业的后面部分，我们会说明在 CPU 或 MPS 上需要做哪些调整。
